좋은 질문입니다. 제안서의 핵심인 "학습 중 편향 제거가 실제로 어떻게 작동하는가"를 단계별로 설명하겠습니다.

---

## Methodology: 학습 과정 중 동적 편향 완화의 작동 원리

### 전체 파이프라인 개요

일반적인 모델 학습에서는 매 이터레이션마다 미니배치를 샘플링하고, 손실을 계산하고, gradient를 구해서 파라미터를 업데이트합니다. 본 방법론은 이 표준 학습 루프에 **하나의 단계를 삽입**합니다. gradient 계산과 파라미터 업데이트 사이에서, 각 훈련 데이터의 "편향되지 않은 검증 데이터에 대한 기여도"를 Shapley value로 측정하고, 이 값에 따라 해당 데이터의 손실 가중치를 즉각 조절하는 것입니다.

---

### Step 1: 사전 준비 — 편향되지 않은 검증 데이터셋(Unbiased Validation Set) 구축

학습을 시작하기 전에, 소규모의 검증 데이터셋 D_val을 준비합니다. 이 데이터셋은 "모델이 궁극적으로 도달해야 하는 공정한 성능"의 기준점(anchor) 역할을 합니다.

핵심은 이 검증 데이터가 인구통계학적으로 균형 잡힌 분포를 가져야 한다는 것입니다. 예를 들어 훈련 데이터에서 남성 데이터가 80%, 여성 데이터가 20%라면, 검증 데이터에서는 50:50 또는 실제 인구 비율에 맞게 구성합니다. 이때 검증 데이터 각각에 "남성/여성" 같은 민감 속성 레이블을 부여할 필요는 없습니다. 데이터셋의 구성 비율 자체가 공정성의 방향을 암묵적으로 인코딩하기 때문입니다.

본 방법론에서 유틸리티 함수는 다음과 같이 정의됩니다:

> **U(z_i; θ_t) = "훈련 데이터 z_i가 t번째 이터레이션에서 검증 데이터 D_val의 loss를 얼마나 줄이는 데 기여하는가?"**

이 정의가 편향 완화의 출발점입니다. 검증 데이터가 균형 잡힌 분포를 가지므로, 검증 loss를 줄이는 데 기여한다는 것은 곧 모든 하위 그룹에 대해 골고루 성능을 개선한다는 의미가 됩니다.

---

### Step 2: 매 이터레이션에서의 기여도 계산 — 1차 근사

모델 학습의 t번째 이터레이션에서, 현재 파라미터 θ_t와 미니배치 내 훈련 데이터 z_i, 그리고 검증 데이터 D_val이 주어졌을 때, In-Run Data Shapley는 다음과 같이 작동합니다.

**Taylor 1차 근사의 핵심 아이디어:** 한 번의 gradient 업데이트로 인한 모델 성능 변화는 충분히 작으므로, 이를 1차 Taylor 전개로 근사할 수 있습니다. 이 근사 하에서, 훈련 데이터 z_i의 Shapley value는 다음과 같이 **해석적으로(closed-form)** 도출됩니다:

> **ϕ_i^(1) ∝ ⟨∇L(z_i; θ_t), ∇L(D_val; θ_t)⟩**

즉, **훈련 데이터 z_i에서의 gradient**와 **검증 데이터에서의 gradient** 사이의 **내적(dot product)**입니다.

이 값의 직관적 의미를 풀어보면 다음과 같습니다:

- **ϕ_i > 0 (양수):** z_i의 gradient가 검증 데이터의 loss를 줄이는 방향과 같은 방향을 가리킵니다. 이 데이터는 모델을 공정한 방향으로 이끄는 "좋은" 데이터입니다.
- **ϕ_i < 0 (음수):** z_i의 gradient가 검증 데이터의 loss를 증가시키는 방향을 가리킵니다. 이 데이터는 모델을 편향된 방향으로 이끄는 "나쁜" 데이터입니다.
- **ϕ_i ≈ 0:** z_i가 검증 데이터의 성능에 별다른 영향을 미치지 않습니다.

**Ghost dot-product를 통한 효율적 계산:** 이 gradient 내적을 나이브하게 계산하면 각 훈련 데이터마다 gradient 벡터(파라미터 수만큼의 차원)를 저장해야 하므로 메모리가 폭발합니다. Ghost dot-product 기법은 추가 gradient 벡터를 인스턴스화하지 않고, 단 **1회의 역전파(backpropagation)**만으로 미니배치 내 모든 훈련 데이터와 검증 데이터 사이의 gradient 내적을 동시에 계산합니다. 이것이 본 방법론이 표준 학습 대비 무시할 수 있는 오버헤드만 추가하는 핵심 기술적 기반입니다.

---

### Step 3: 2차 근사를 통한 과잉 대표 자동 교정

1차 근사만으로도 기본적인 편향 완화가 가능하지만, 2차 근사는 추가적인 **자동 균형 메커니즘**을 제공합니다.

2차 Taylor 전개에서는 훈련 데이터 z_i의 Shapley value에 다른 훈련 데이터들과의 **상호작용 항**이 추가됩니다:

> **ϕ_i^(2) = ϕ_i^(1) − (보정 항)**
> 
> 보정 항 ∝ Σ_{j≠i} ⟨∇L(z_i; θ_t), H · ∇L(z_j; θ_t)⟩

여기서 H는 Hessian 행렬이고, 이 보정 항은 **"z_i가 다른 훈련 데이터 z_j들과 얼마나 유사한 정보를 제공하는가"**를 측정합니다.

이것이 편향 완화에서 작동하는 원리는 다음과 같습니다. 다수 그룹(예: 남성)의 데이터가 훈련셋의 80%를 차지한다고 가정합니다. 이 그룹의 데이터들은 서로 유사한 특성을 공유하므로, gradient 방향이 비슷합니다. 2차 상호작용 항에서 이 유사한 데이터들 사이의 누적 보정이 커지면서, 각 개별 데이터의 Shapley value가 수학적으로 분산되고 감소합니다. 반면 소수 그룹(예: 여성)의 데이터는 상대적으로 적으므로 상호작용 항의 누적이 작고, 개별 데이터의 Shapley value가 상대적으로 높게 유지됩니다.

결과적으로, **명시적인 그룹 레이블이나 균형 제약 없이도**, 다수 그룹의 데이터에는 낮은 기여도가, 소수 그룹의 데이터에는 높은 기여도가 자동으로 할당됩니다. 이 계산은 ghost gradient-Hessian-gradient product 기법으로 **추가 1회의 역전파**(총 2회)만으로 효율적으로 수행됩니다.

---

### Step 4: Shapley Value를 샘플별 가중치로 변환

계산된 Shapley value ϕ_i를 손실 함수의 가중치 w_i로 변환하는 과정이 필요합니다. 이 변환 함수의 설계가 알고리즘의 안정성에 직결됩니다.

**가중치 변환 함수:** 원시 Shapley value를 직접 가중치로 사용하면 음수 가중치나 극단적인 값이 발생할 수 있으므로, 정규화와 클리핑을 적용합니다:

> **w_i = clip(softmax(ϕ_i / τ), w_min, w_max)**

여기서 τ는 온도(temperature) 하이퍼파라미터로, 가중치 분포의 선명도를 조절합니다. τ가 크면 가중치가 균등에 가까워지고(보수적 조절), τ가 작으면 극단적으로 차이가 벌어집니다(공격적 조절). w_min과 w_max는 어떤 데이터도 완전히 무시되거나 과도하게 증폭되지 않도록 하는 안전 장치입니다.

**지수 이동 평균(EMA)을 통한 안정화:** 매 이터레이션의 Shapley value는 미니배치 의존적 노이즈를 포함하므로, 누적된 Shapley value의 지수 이동 평균을 사용하여 가중치를 안정화합니다:

> **Φ_i^(t) = α · Φ_i^(t-1) + (1-α) · ϕ_i^(t)**

이 누적 값을 기반으로 가중치를 산출하면, 한 이터레이션의 이상치에 의해 가중치가 급변하는 것을 방지할 수 있습니다.

---

### Step 5: 가중 손실을 통한 파라미터 업데이트

최종적으로, t번째 이터레이션의 파라미터 업데이트는 표준 SGD가 아닌 **가중 SGD**로 수행됩니다:

> **θ_{t+1} = θ_t − η · Σ_i w_i · ∇L(z_i; θ_t)**

편향을 강화하는 데이터(ϕ_i < 0)는 낮은 w_i를 받아 gradient에 미치는 영향이 축소되고, 공정성에 기여하는 데이터(ϕ_i > 0)는 높은 w_i를 받아 gradient에 미치는 영향이 증폭됩니다. 이 과정이 매 이터레이션마다 반복되면서, 모델은 자연스럽게 검증 데이터가 대표하는 "편향되지 않은 분포"를 향해 학습이 유도됩니다.

---

### 전체 알고리즘 요약

하나의 이터레이션에서 일어나는 일을 순서대로 정리하면 다음과 같습니다:

1. 훈련 데이터 미니배치 B_train과 검증 데이터 미니배치 B_val을 샘플링한다.
2. B_train의 각 데이터에 대한 개별 loss gradient와 B_val에 대한 집합 loss gradient를 계산한다 (표준 forward + backward pass).
3. Ghost dot-product(1차) 또는 ghost gradient-Hessian-gradient product(2차) 기법으로, 추가 1~2회의 역전파만으로 각 훈련 데이터의 Shapley value ϕ_i를 계산한다.
4. 누적 Shapley value를 EMA로 업데이트하고, 이를 가중치 변환 함수에 통과시켜 샘플별 가중치 w_i를 산출한다.
5. 가중 손실 Σ w_i · L(z_i; θ_t)의 gradient로 파라미터를 업데이트한다.

이 과정에서 **모델 재학습은 전혀 발생하지 않으며**, 표준 학습 루프에 3~4번 단계가 삽입되는 것이 유일한 차이입니다.

---

### 왜 이 과정이 편향을 제거하는가? — 직관적 설명

편향된 훈련 데이터로 학습하면, 모델의 gradient는 다수 그룹의 패턴을 더 잘 학습하는 방향으로 편향됩니다. 균형 잡힌 검증 데이터의 gradient는 모든 그룹에 대해 골고루 성능을 개선하는 방향을 가리킵니다. 이 두 gradient 사이의 내적을 통해, "다수 그룹에만 편향적으로 기여하는 훈련 데이터"는 검증 gradient와의 정렬도가 낮아 낮은 Shapley value를 받게 되고, "소수 그룹을 포함하여 전체적으로 기여하는 훈련 데이터"는 높은 Shapley value를 받게 됩니다.

이를 비유하자면, 검증 데이터는 "나침반"이고 Shapley value는 "각 훈련 데이터가 이 나침반이 가리키는 방향과 얼마나 같은 방향으로 모델을 밀고 있는가"를 측정하는 것입니다. 나침반과 반대 방향으로 미는 데이터의 힘을 줄이고, 같은 방향으로 미는 데이터의 힘을 키우면, 모델은 자연스럽게 나침반이 가리키는 공정한 방향으로 수렴하게 됩니다.
